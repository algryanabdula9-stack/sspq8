# SSP.Q8 — نموذج تجريبي مُعاد البناء

## التشغيل
افتح `index.html` في متصفح Edge أو Chrome. جميع البيانات تُحفظ في `localStorage` على نفس المتصفح فقط.

## تجربة الأدوار
1. العميل: من `portal.html` أدخل البريد ورقم الهاتف؛ يظهر كود OTP على الشاشة في النسخة التجريبية.
2. الأدمن: `admin.html` — البريد `admin@sspq8.com` وكلمة المرور `sspq8-admin-2026`.
3. الشركة: أنشئ حساباً من تبويب الشركة، ثم فعّل اشتراكها من لوحة الأدمن.

## تدفق العمل الصحيح
- العميل يدفع لـ SSP.Q8 مقابل تحليل أو نموذج أولي فقط.
- الأدمن يرفع الدراسة والنموذج وملفات التسليم.
- العميل ينزل ملفاته ووثيقة توثيق تحمل رقم الطلب والتاريخ.
- يمكنه إرسال الحزمة لكل الشركات، حسب التخصص، أو لعدة شركات محددة.
- الشركات تقدم عروضاً تقريبية فقط.
- بعد اختيار العرض، يظهر التواصل للطرفين ويكون الاتفاق والدفع والتنفيذ النهائي خارج SSP.Q8.

## تنبيه
هذه نسخة عرض محلية وليست صالحة لبيانات حقيقية أو دفع حقيقي. النسخة الإنتاجية تحتاج Backend، قاعدة بيانات، تخزين ملفات، OTP فعلي، بوابة دفع، وصلاحيات آمنة.
